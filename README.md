# 🧃 findmydietcoke

I could not find Diet Coke on Zepto. I have a computer science degree.

---

## 🚀 Features

* **Real-time Headless Scraping**: Spawns an entire invisible Chromium browser — a fully functional ghost computer inside your computer — to check if a beverage exists. The beverage costs ₹40. The electricity to run this probably costs more.

* **Precise Location Targeting**: Zepto's dropdown blocked normal clicks, so I wrote coordinate math to physically aim at the correct pixel and click it like a sniper. For Diet Coke. I did this.

* **Client-side Fetch Interception**: Injects JavaScript directly into Zepto's brain to steal their internal API responses before the page can hide them. This is technically impressive. It is also completely deranged. Both things are true.

* **Smart Caching Layer**: Caches results for 5 minutes to avoid spinning up the ghost browser too often. The ghost browser exists because I couldn't check an app. I am caching the results of my own failure.

* **Hacker Aesthetic UI**: Black background, monospace font. If the app isn't going to save me time, it will at least make me look cool while it wastes it.

---

## 🛠️ Tech Stack

**Next.js 14, TypeScript, Playwright.**

I used a full-stack TypeScript framework with a headless browser automation engine typically used to test enterprise software, to find a soft drink. The soft drink was out of stock. It has been out of stock many times since. I have not stopped running the app.

---

## ⚙️ How It Works

1. You type your address.
2. A secret browser wakes up, opens Zepto, and navigates it like a haunted intern.
3. It types "diet coke", scrolls down, and intercepts the raw network responses before they can escape.
4. It reports back: IN_STOCK or OUT_OF_STOCK.
5. You could have opened the Zepto app and found this out in 4 seconds.
6. My app takes longer than 4 seconds.
7. Zepto delivers in 10 minutes regardless.
8. None of this was necessary.
